# SDD Spec Validation Skill

Formal specification validation skill for a **Spec Driven Development** flow.

## Objective

Act as a gate prior to implementation, verifying that the artifacts generated by the previous skills are complete, coherent, traceable and secure enough to allow execution.

## Place in the flow

```text
sdd-functional-spec
  → sdd-technical-spec
  → sdd-api-contract, if applicable
  → sdd-migration-rollback, if applicable
  → sdd-security-permissions-review, if applicable
  → sdd-spec-validation
  → sdd-implementation
```

## Possible decisions

- `READY_FOR_IMPLEMENTATION`: Deployment can begin.
- `NEEDS_REFINEMENT`: you must return to a previous skill.
- `BLOCKED`: there is a critical block and the orchestrator or a human decision must intervene.

##Main artifacts

```text
spec-validation-report.md
spec-readiness-checklist.md
traceability-matrix.md
implementation-gate-decision.md
spec-remediation-plan.md
```

## Recommended use

Always use this skill before running `sdd-implementation`. The orchestrator must invoke it as a mandatory gate when there is any functional, technical, data, permissions or API change.
