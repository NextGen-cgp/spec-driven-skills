# Final Review Checklist

##Entry Gate

- [ ] `implementation-report.md` exists.
- [ ] `test-report.md` exists.
- [ ] `acceptance-validation-report.md` exists.
- [ ] `deviation-log.md` has been reviewed.
- [ ] Required security artifacts are present, if applicable.

## Spec Compliance

- [ ] All acceptance criteria are traced.
- [ ] All business rules are accounted for.
- [ ] No unapproved scope creep detected.
- [ ] Deviations are documented and acceptable.

## Tests

- [ ] Functional tests are documented.
- [ ] Regression tests are documented.
- [ ] Contract tests exist if API/data changed.
- [ ] Authorization tests exist if permissions changed.
- [ ] Migration tests exist if persistence changed.

## Security

- [ ] Backend authorization is enforced where required.
- [ ] Roles cannot perform unauthorized operations.
- [ ] Data exposure is controlled.
- [ ] State transitions are protected.
- [ ] Error responses are safe.

##Quality

- [ ] Code follows project patterns.
- [ ] No major duplication.
- [ ] Maintainability is acceptable.
- [ ] Operational risks are documented.

##Exit

- [ ] Final decision issued.
- [ ] Next route issued.
- [ ] Required actions documented.
